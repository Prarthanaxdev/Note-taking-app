# Proposal — AB-1013: Search UI (Frontend)

## Summary

Add the full-text search experience to the React frontend:

1. A **search bar** in the `NotesListPage` header that navigates to `/search?q=<term>` on submit.
2. A **SearchPage** at `/search` that reads `q` + `page` from URL params, calls `GET /notes/search`,
   renders highlighted results (with `<mark>` tag support), a loading skeleton, a no-results empty
   state, and full pagination controls.
3. A **`useSearch` hook** (TanStack Query) that wraps the search API call.

The backend FTS endpoint (`GET /notes/search`) was implemented in AB-1007 and is already live.
No changes to `packages/shared` are needed — `SearchResult`, `PaginationMeta`, and
`SearchQuerySchema` already exist there.

## Key Design Decisions

### URL-driven search state
`SearchPage` uses `useSearchParams()` to read and write `q` and `page`. This makes search results
bookmarkable, shareable, and back-button-friendly. TanStack Query key includes both params, so
distinct queries are cached separately.

### `dangerouslySetInnerHTML` for headline
The `headline` field returned by `GET /notes/search` is HTML generated by PostgreSQL's
`ts_headline()` with `StartSel=<mark>` and `StopSel=</mark>`. It comes from our own server
(not user-controlled) so `dangerouslySetInnerHTML` is acceptable per AGENTS.md §11
("No raw `dangerouslySetInnerHTML` except search headlines").

### Search triggered by Enter / submit only
The search bar in both `NotesListPage` header and `SearchPage` fires only on form submit
(Enter key or search button click). No debounced live search to avoid hammering the FTS
endpoint on every keystroke. On `SearchPage`, editing the input updates local state; pressing
Enter updates the URL params (which triggers a new query).

### useSearch enabled guard
`enabled: params.q.trim().length > 0` prevents an API call when `q` is empty. When the user
navigates to `/search` with no query, the page shows the input with a "Enter a search term…"
prompt instead of making a request.

### Pagination component reuse
Reuses the `Pagination` component from AB-1011 (`apps/web/src/components/notes/Pagination.tsx`).
Page number lives in the URL as `page` param, reset to 1 on every new query.

### NotesListPage search bar placement
Added to the existing header row alongside the "My Notes" heading, left-aligned. It is a
controlled `<input>` with a `<Search>` lucide icon and a submit handler. On submit it calls
`navigate('/search?q=<term>')` — it does NOT call the search API directly.

## Interface Contracts

```typescript
// hooks/useSearch.ts
export interface SearchParams {
  q: string;
  page: number;
  limit: number;
}

export function useSearch(params: SearchParams) {
  return useQuery<{ data: SearchResult[]; meta: PaginationMeta }>({
    queryKey: ['search', params],
    queryFn: () =>
      apiClient
        .get<{ data: SearchResult[]; meta: PaginationMeta }>('/notes/search', { params })
        .then((r) => r.data),
    enabled: params.q.trim().length > 0,
  });
}
```
