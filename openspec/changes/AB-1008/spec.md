## 1. Behavioral Requirements

### 1.1 Shared Schema Change

| ID | Requirement |
|---|---|
| SHARE-REQ-01 | `CreateShareSchema.expiresAt` SHALL use `.refine(d => new Date(d) > new Date(), ...)` to reject past datetimes. A missing `expiresAt` field is still valid (field is optional). |
| SHARE-REQ-02 | A past `expiresAt` value SHALL return `400 VALIDATION_ERROR`. |

### 1.2 `POST /notes/:id/share` — Create Share Link

| ID | Requirement |
|---|---|
| SHARE-REQ-03 | The endpoint SHALL require a valid Bearer JWT. Missing or invalid token → `401 UNAUTHORIZED`. |
| SHARE-REQ-04 | The service SHALL verify that the note exists, belongs to the authenticated user, and is not soft-deleted. Failure → `404 NOT_FOUND`. |
| SHARE-REQ-05 | The share token SHALL be a UUID generated by PostgreSQL `@default(uuid())`. It is not generated in application code. |
| SHARE-REQ-06 | A note MAY have multiple simultaneous active share links (BR-SHARE-02). Creating a second link for the same note is valid. |
| SHARE-REQ-07 | On success, return `201` with the full `ShareLink` object. The response does NOT include a constructed public URL — the token is sufficient for the frontend to construct one. |
| SHARE-REQ-08 | `ShareLink` response shape: `{ id, noteId, userId, token, expiresAt: string|null, revokedAt: string|null, viewCount: number, createdAt: string }`. All dates are ISO 8601 strings. |

### 1.3 `GET /notes/:id/shares` — List Share Links

| ID | Requirement |
|---|---|
| SHARE-REQ-09 | The endpoint SHALL require a valid Bearer JWT. |
| SHARE-REQ-10 | The service SHALL verify note ownership (same check as SHARE-REQ-04). |
| SHARE-REQ-11 | The endpoint SHALL return ALL share links for the note — including revoked and expired ones — ordered by `createdAt DESC`. |
| SHARE-REQ-12 | An empty array `[]` is a valid response when no links exist. |

### 1.4 `DELETE /shares/:shareId` — Revoke Share Link

| ID | Requirement |
|---|---|
| SHARE-REQ-13 | The endpoint SHALL require a valid Bearer JWT. |
| SHARE-REQ-14 | The service SHALL look up the share link by `{ id: shareId, userId }`. A missing link or mismatched userId → `404 NOT_FOUND` (BR-SHARE-04 enumeration prevention). |
| SHARE-REQ-15 | Revocation SHALL set `revokedAt = now()` via a single `prisma.shareLink.update()`. It is a permanent operation — no re-activate (BR-SHARE-07). |
| SHARE-REQ-16 | On success, return `204 No Content`. |

### 1.5 `GET /public/notes/:token` — Public Read

| ID | Requirement |
|---|---|
| SHARE-REQ-17 | The endpoint SHALL NOT have `authenticate` middleware — it is fully public (no Bearer token required). |
| SHARE-REQ-18 | The service SHALL look up the `ShareLink` by `token` (unique index). Token not found → `404`. |
| SHARE-REQ-19 | The service SHALL validate: `revokedAt IS NULL` AND (`expiresAt IS NULL` OR `expiresAt > now()`). A revoked or expired link → `404` (BR-SHARE-03, BR-SHARE-04). |
| SHARE-REQ-20 | The service SHALL validate that the underlying note is not soft-deleted (`note.deletedAt === null`). Soft-deleted note → `404` (BR-SHARE-08). |
| SHARE-REQ-21 | The service SHALL atomically increment `viewCount` using `{ increment: 1 }` — never read-modify-write (BR-SHARE-05, AGENTS.md constraint). |
| SHARE-REQ-22 | On success, return `200` with ONLY `{ title: string, content: object | null }`. No tags, no share link metadata, no userId (BR-SHARE-06). |

### 1.6 Cross-Cutting Constraints

| ID | Requirement |
|---|---|
| SHARE-REQ-23 | `userId` is always sourced from `req.user.id` at the route layer — never from body or query params. |
| SHARE-REQ-24 | Authorization is enforced in the service layer via `findFirst({ where: { id, userId } })`. |
| SHARE-REQ-25 | Cross-user access to notes or share links SHALL return `404`, never `403`. |

## 2. Test Scenarios

### 2.1 Unit Tests — `shares.service.test.ts`

> Mock `../../lib/prisma.js` with:
> ```typescript
> prisma: {
>   note: { findFirst: vi.fn() },
>   shareLink: { create: vi.fn(), findMany: vi.fn(), findFirst: vi.fn(), findUnique: vi.fn(), update: vi.fn() }
> }
> ```

| Test ID | Function | Scenario | Expected |
|---|---|---|---|
| SHARE-UT-01 | `createShareLink` | Note exists and belongs to user | Returns `ShareLink`; `shareLink.create` called once |
| SHARE-UT-02 | `createShareLink` | `note.findFirst` returns `null` | Throws `AppError('NOT_FOUND', ..., 404)` |
| SHARE-UT-03 | `createShareLink` | `expiresAt` provided | `shareLink.create` called with `expiresAt: new Date(...)` |
| SHARE-UT-04 | `createShareLink` | No `expiresAt` | `shareLink.create` called with `expiresAt: null` |
| SHARE-UT-05 | `listShareLinks` | Note owned by user, 2 links exist | Returns array of 2 `ShareLink` objects with ISO date strings |
| SHARE-UT-06 | `listShareLinks` | Note not found | Throws `AppError('NOT_FOUND', ..., 404)` |
| SHARE-UT-07 | `listShareLinks` | No share links exist | Returns `[]` |
| SHARE-UT-08 | `revokeShareLink` | Link found with matching userId | Calls `shareLink.update({ data: { revokedAt: expect.any(Date) } })`; returns void |
| SHARE-UT-09 | `revokeShareLink` | `shareLink.findFirst` returns `null` | Throws `AppError('NOT_FOUND', ..., 404)` |
| SHARE-UT-10 | `getPublicNote` | Active link, note not soft-deleted | Returns `{ title, content }`; `shareLink.update` called with `{ increment: 1 }` |
| SHARE-UT-11 | `getPublicNote` | Token not found (`findUnique` returns `null`) | Throws `AppError('NOT_FOUND', ..., 404)`; `update` NOT called |
| SHARE-UT-12 | `getPublicNote` | `revokedAt` is set | Throws `AppError('NOT_FOUND', ..., 404)` |
| SHARE-UT-13 | `getPublicNote` | `expiresAt` is in the past | Throws `AppError('NOT_FOUND', ..., 404)` |
| SHARE-UT-14 | `getPublicNote` | Note `deletedAt` is set | Throws `AppError('NOT_FOUND', ..., 404)` |
| SHARE-UT-15 | `getPublicNote` | `expiresAt` is `null` (never expires) | Returns `{ title, content }` (null expiry = permanent link) |

## 4. Error Code Reference

| Code | Status | Trigger |
|---|---|---|
| `NOT_FOUND` | 404 | Note not found or not owned; share link not found or not owned; expired/revoked/deleted on public access |
| `VALIDATION_ERROR` | 400 | Past `expiresAt`; malformed datetime |
| `UNAUTHORIZED` | 401 | Missing or invalid Bearer JWT on protected routes |
